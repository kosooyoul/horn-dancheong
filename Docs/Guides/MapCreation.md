# 맵 제작 가이드

## 🎯 개요

이 가이드는 Horn Dancheong 프로젝트에서 새로운 배틀맵을 제작하는 방법을 설명합니다.

## 📋 사전 준비

### 필요한 파일
1. **TILES.json** - 타일 정의 (이미 준비됨)
2. **OBJECTS.json** - 오브젝트 정의 (이미 준비됨)  
3. **새 맵 JSON 파일** - 제작할 맵 데이터

### 권장 도구
- JSON 편집기 (VS Code, Sublime Text 등)
- 맵 칩 번호 계산기 (아래 제공)
- 그리드 종이 또는 디지털 그리드 툴

## 🗺️ 맵 설계 단계

### 1단계: 컨셉 설계
맵의 기본 컨셉을 결정합니다.

**질문 체크리스트:**
- [ ] 맵의 크기는? (권장: 10x10 ~ 25x25)
- [ ] 전투 스타일은? (근접, 원거리, 혼합)
- [ ] 주요 지형 특징은? (벽, 장애물, 열린 공간)
- [ ] 대칭성이 필요한가? (PvP라면 대칭 권장)
- [ ] 특별한 기믹이 있는가? (포탈, 함정 등)

**예시 컨셉:**
```
맵명: "Forest Clearing"
크기: 15x15
컨셉: 숲 속 빈터, 중앙은 열린 공간, 가장자리에 나무
특징: 비대칭, 엄폐물 다수, 상호작용 요소 포함
```

### 2단계: 그리드 스케치
종이나 디지털 툴로 맵을 스케치합니다.

**기호 표기 예시:**
```
. = 빈 공간 (타일 없음)
# = 벽
o = 바닥
T = 나무  
R = 바위
C = 상자
```

**샘플 스케치 (7x7):**
```
# # # # # # #
# o o T o o #
# o R o R o #
# T o C o T #  
# o R o R o #
# o o T o o #
# # # # # # #
```

### 3단계: 맵 칩 번호 할당
각 위치에 맵 칩 번호를 할당합니다.

## 🔢 맵 칩 번호 계산

### 계산 공식
```
맵 칩 번호 = (오브젝트 ID << 8) | 타일 ID
```

### 자주 사용되는 조합

#### 기본 타일만
```
0     = 빈 공간
1     = 바닥
255   = 벽  
```

#### 타일 + 오브젝트 조합
```
513   = 바닥 + 나무 (1 | (2 << 8))
769   = 바닥 + 덤불 (1 | (3 << 8)) 
1025  = 바닥 + 상자 (1 | (4 << 8))
1281  = 바닥 + 배럴 (1 | (5 << 8))
2305  = 바닥 + 횃불 (1 | (9 << 8))
```

### 계산 도우미 테이블

| 타일 ID | 오브젝트 ID | 맵 칩 번호 | 설명 |
|---------|-------------|-----------|------|
| 1 | 0 | 1 | 바닥만 |
| 255 | 0 | 255 | 벽만 |
| 1 | 1 | 257 | 바닥 + 바위 |
| 1 | 2 | 513 | 바닥 + 나무 |
| 1 | 3 | 769 | 바닥 + 덤불 |
| 1 | 4 | 1025 | 바닥 + 상자 |
| 1 | 5 | 1281 | 바닥 + 배럴 |
| 2 | 10 | 2562 | 중앙타일 + 스폰포인트 |

## 📝 JSON 파일 작성

### 기본 템플릿
```json
{
  "mapName": "Your Map Name",
  "description": "맵에 대한 설명",
  "width": 15,
  "height": 15,
  "tiles": [
    // 맵 칩 번호들을 width x height 개수만큼 나열
    // row-major order (좌→우, 위→아래)
  ],
  "tileTypes": [
    // 호환성을 위한 기본 타일 정의 (변경하지 마세요)
  ]
}
```

### 실제 예시 (3x3 맵)
```json
{
  "mapName": "Tiny Arena",
  "description": "3x3 테스트 맵",
  "width": 3,
  "height": 3,
  "tiles": [
    255, 255, 255,
    255, 1025, 255,  
    255, 255, 255
  ],
  "tileTypes": [
    {"id": 0, "name": "Empty", "colorHex": "#000000"},
    {"id": 1, "name": "Floor", "colorHex": "#E6E6FA"},
    {"id": 255, "name": "Wall", "colorHex": "#696969"}
  ]
}
```

## 🎨 맵 설계 패턴

### 대칭 맵 (PvP용)
```json
{
  "mapName": "Symmetrical Arena",
  "width": 11,
  "height": 11,
  "tiles": [
    // 중앙축 기준으로 대칭 배치
    // 플레이어 시작 위치도 대칭으로
  ]
}
```

### 비대칭 맵 (PvE용)
```json
{
  "mapName": "Forest Encounter", 
  "width": 20,
  "height": 15,
  "tiles": [
    // 자연스러운 지형 배치
    // 다양한 전술적 선택지 제공
  ]
}
```

### 특수 기믹 맵
```json
{
  "mapName": "Crystal Cave",
  "description": "크리스탈과 포탈이 있는 동굴",
  "tiles": [
    // 포탈 (id: 11) 배치: 2817 = 1 | (11 << 8)
    // 크리스탈 (id: 8) 배치: 2049 = 1 | (8 << 8)
  ]
}
```

## 🛠️ 디버깅 팁

### 일반적인 오류들

#### 배열 크기 불일치
```
오류: tiles 배열 길이가 width × height와 다름
해결: 배열 개수를 정확히 맞춰주세요
```

#### 잘못된 맵 칩 번호
```
경고: 정의되지 않은 타일/오브젝트 ID
해결: TILES.json, OBJECTS.json에 정의된 ID만 사용
```

#### 접근 불가능한 영역
```
문제: 일부 영역이 벽으로 막혀 접근 불가
해결: 모든 바닥 타일이 연결되도록 경로 확보
```

### 디버그 도구 활용
```csharp
// 맵 정보 출력
Debug.Log($"맵 크기: {battleScript.GetMapSize()}");
Debug.Log($"(5,3) 이동 가능: {battleScript.IsWalkable(5, 3)}");
Debug.Log($"(5,3) 타일 ID: {battleScript.GetTileType(5, 3)}");
Debug.Log($"(5,3) 오브젝트 ID: {battleScript.GetObjectType(5, 3)}");
```

## 🎯 맵 설계 모범 사례

### 전술적 다양성
- **엄폐물**: 원거리 공격 차단용
- **좁은 통로**: 근접 전투 유도
- **열린 공간**: 기동성 중시 전투
- **높이 차이**: 시야 및 사거리 변화 (미래 확장)

### 밸런스 고려사항
- **시작 위치**: 공정한 거리 및 지형
- **핵심 위치**: 중앙 제고권의 가치
- **측면 공격**: 우회 경로 존재 여부
- **상호작용 요소**: 전략적 선택의 의미

### 시각적 매력
- **리듬감**: 반복과 변화의 조화
- **초점**: 시각적 랜드마크 배치
- **흐름**: 자연스러운 이동 경로
- **색상**: 타일/오브젝트 색상 조화
