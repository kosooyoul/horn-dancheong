# 맵 시스템 스펙 문서

## 📋 개요

Horn Dancheong 프로젝트의 맵 시스템은 2D 그리드 기반의 전술 전투를 위한 배틀맵을 관리합니다.

## 🏗️ 시스템 아키텍처

### 핵심 컴포넌트
1. **BattleScript** - 맵 로딩 및 관리 메인 클래스
2. **TILES.json** - 바닥 타일 정의 파일
3. **OBJECTS.json** - 오브젝트 정의 파일
4. **맵 데이터 파일들** - 실제 맵 레이아웃 정의

### 데이터 흐름
```
맵 JSON 파일 → BattleScript → 타일/오브젝트 생성 → Unity 씬
      ↑              ↑
TILES.json    OBJECTS.json
```

## 🔢 맵 칩 번호 체계

### 구조
```
32비트 정수 = (ObjectID << 8) | TileID
```

### 비트 구성
- **0x0000FF (하위 8비트)**: 타일 ID
- **0x00FF00 (중위 8비트)**: 오브젝트 ID  
- **0xFF0000 (상위 8비트)**: 예약됨 (미래 확장용)

### 예시
```
값: 1281 (0x0501)
→ 타일 ID: 1 (Floor)
→ 오브젝트 ID: 5 (Barrel)
```

## 📐 좌표 시스템

### 그리드 좌표
- **원점**: 맵의 좌상단 (0, 0)
- **X축**: 우측 방향 증가
- **Z축**: 아래쪽 방향 증가 (Unity 3D 공간에서)

### 월드 좌표 변환
```csharp
Vector3 worldPos = new Vector3(
    gridX * cubeSpacing - offsetX,
    0f,
    gridZ * cubeSpacing - offsetZ
);
```

## 🎯 주요 기능

### 맵 로딩
- JSON 파일에서 맵 데이터 로딩
- 타일 및 오브젝트 정의 로딩
- 런타임 맵 교체 지원

### 맵 생성
- 타일별 바닥 오브젝트 생성
- 오브젝트별 3D 모델 배치
- 계층적 씬 구조 관리

### 게임플레이 지원
- 이동 가능 여부 판정
- 이동 비용 계산
- 시야 차단 검사
- 상호작용 가능 오브젝트 관리

## 🔧 설정 가능 옵션

### Inspector 설정
- `floorCubePrefab`: 바닥 타일 프리팹
- `cubeSpacing`: 타일 간격 (기본값: 1.0f)
- `mapFolderPath`: 맵 데이터 폴더 경로
- `defaultMapName`: 기본 로딩 맵 이름
- `loadFromJSON`: JSON 사용 여부

### 폴백 시스템
- JSON 로딩 실패 시 하드코딩된 기본 맵 사용
- 정의되지 않은 타일/오브젝트에 대한 기본 처리

## 🚀 성능 최적화

### 메모리 관리
- 맵 교체 시 기존 오브젝트 자동 해제
- 딕셔너리 기반 빠른 타일/오브젝트 정보 조회

### 렌더링 최적화
- 부모 Transform을 통한 배치 최적화
- 프리팹 풀링 지원 (확장 가능)

## 🔄 확장성

### 미래 확장 계획
- 다층 맵 지원 (상위 8비트 활용)
- 동적 맵 변경 시스템
- 맵 에디터 툴 연동
- 네트워크 동기화 지원
